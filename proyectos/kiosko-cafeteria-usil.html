Kiosko Cafetería USIL

Sistema web para optimizar pedidos en una cafetería universitaria mediante un kiosko público y paneles administrativos por rol.

Problema

Contexto del proyecto

En una cafetería con alta demanda, el proceso manual de pedidos puede generar colas, errores de atención, poca visibilidad del stock y dificultad para controlar pedidos por estado.

Solución

Qué construí

Desarrollé un sistema web con kiosko público, catálogo, carrito, pago simulado, comprobante, paneles por rol, control de stock, pedidos y reportes gerenciales.

Mi participación

Rol y aporte

Participé en la construcción del sistema completo: interfaz, lógica de pedidos, backend, estructura de datos, paneles y documentación técnica del proyecto.

Funciones

Características principales

  • Kiosko público con catálogo, carrito y flujo de compra.
  • Panel administrativo para productos, usuarios, pedidos y reportes.
  • Paneles diferenciados para cafetería y cocina.
  • Control de stock y actualización de productos.
  • API REST con autenticación JWT y persistencia en PostgreSQL.
